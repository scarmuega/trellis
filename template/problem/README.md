# Founding map

The domain's invariant layer: market needs and the jobs-to-be-done they imply.
Technology-free by rule — no strategy vocabulary, no chosen means, no product or
stack names (the lint flags them). A pivot rewrites `strategy/` and the subdomains
it induced; this map endures.

- Needs, one anchored section each (`## N-{slug}`) — strategies ref them
- Market: segmentation, competitive landscape, substitutes
- Personas and their as-is journeys, as anchored sections below
- Subdomains: one file per subdomain in this directory, each carrying the
  `induced-by:` edges naming the strategy that put it there — classification lives
  on the edge and drives automation policy

## N-{slug}

The need, stated so it survives any pivot in means — external, slow-changing,
independent of this business. The jobs-to-be-done it implies. One `## N-{slug}`
section per need.

## Personas

### {persona}

Who they are / needs (ref the `## N-` anchors) / context / willingness to pay.

**As-is journey:** how they experience the problem today, where it hurts.
