# 0007 — Rationale as grounded premises with a deletion corollary

## Context
The framework needed its premises stated: why this structure, why now, why DDD, why
one shape for humans and agents.

## Decision
Seven premises adopted, each carrying an explicit *grounds* mapping to the
framework elements it justifies: (1) business as search for problem-solution fit
under economic constraint; (2) imperfect information — the framework is an
epistemic machine; (3) agents collapse execution AND ceremony cost — documentation
is the execution medium; (4) under agents the whole solution space is
software-shaped, licensing DDD wholesale; (5) versioned natural language is the
shared human/agent medium; (6) structure substitutes for memory — agents are
stateless; (7) delegated execution requires explicit authority.

## Consequences
Corollary adopted: any framework element grounded by no premise is a deletion
candidate. The rationale is an audit tool, not decoration.
